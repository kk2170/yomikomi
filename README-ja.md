# YomiKomi

YomiKomi は、**「流れていく情報を、自分の手元に取り戻す」** ためのローカルRSSリーダーです。

ただ新着を眺めるだけでは終わりません。  
気になった記事は**その場で丸ごと保存**し、あとでネットの向こう側ではなく、**自分のローカル環境でじっくり読む**。  
タイムラインに飲まれず、アルゴリズムに急かされず、読みたいものを、自分の速度で読むためのアプリです。

## いま目指しているもの

- RSS / Atom フィードを登録して、自分だけの購読環境を作る
- 記事一覧をまとめて読み、気になった記事を見逃さない
- 重要な記事は本文ごとローカルDBへ保存する
- 保存した記事を、ローカルの読書ビューで落ち着いて読む
- 必要なら Wayback Machine へ魚拓取得を飛ばす

## いま出来ること

- Electron + React + TypeScript ベースのデスクトップアプリ
- RSS / Atom フィード URL の登録・削除
- フィード情報の**ローカルDB保存**
- 記事一覧の取得と表示
- 気になった記事の**全文抽出とローカル保存**
- 保存済み記事の**ローカル閲覧**
- ダーク / ライト / システム追従の**表示モード切替**
- Ocean / Forest / Sunset / Paper の**スタイル選択**
- SQLiteデータベースの**エクスポート**
- 元記事を外部ブラウザで開く
- Wayback Machine を使った魚拓取得導線

## なぜYomiKomiか

Webの記事は、ある日突然消えます。  
SNSの流れは速すぎます。  
ブックマークだけでは、あとで読まないことも多いです。

だからYomiKomiは、

- **購読する**
- **見つける**
- **保存する**
- **手元で読む**

この流れを一つのデスクトップアプリに閉じ込めます。

## 今後の構想

- 記事URLから対応するRSS/Atomフィード候補を探索
  - MVP方針としては、記事ページ内の `rel="alternate"` を読む方法と、`/feed` や `rss.xml` などの定番パス探索が有力
  - ただし、これは **常に唯一の正解を返せる機能ではなく、フィード候補を高確率で見つける機能** として考えるのが現実的
- 保存済み記事の検索
- タグ付け・後で読む整理
- オフライン読書体験の強化

## 思想

YomiKomi は、情報を消費するためではなく、**情報と付き合い直すため** のアプリです。  
読む価値があるものを、読める場所に。  
残したいものを、残せる形に。  

そのための小さな拠点を、ローカルに作ります。
